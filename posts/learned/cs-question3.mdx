**질문1.** Javascript의 호이스팅에 대해 설명해주세요.

**답변1.** 호이스팅이란 코드의 실행 컨텍스트 생성 시 변수 선언, 함수 선언이 해당 스코프의 최상단으로 끌어올려지는
것과 같은 현상을 의미합니다. 이러한 현상으로 선언 키워드에 따라 코드 선언부 도달 전 해당 값을 참조 혹은 호출할 수도 있습니다.
함수 선언식은 실행 컨텍스트에 함수가 포함되므로 전역에서 참조 가능합니다. 함수 표현식으로 사용하면 함수가 할당된 변수명만
포함되므로 실행 컨텍스트를 간결하게 만들 수 있습니다.

- 함수 선언 방식에 따라 호출 가능 여부, 실행 동작 차이점
- 실행 컨텍스트란, 코드가 실행될 때 필요한 환경 정보를 담고있는 객체
- 자바스크립트는 코드 실행 전, 인터프리터가 변수, 함수 공간을 렉시컬 환경 내 메모리에 추가한다.
- 장점은 코드의 유연성 증가, 단점은 불필요한 초기화값(var undefined), 함수 선언 호이스팅

**[Keyword]** 인터프리터, 선언, 할당

**질문2.** 동기와 비동기의 차이에 대해 설명해주시고 비동기프로그래밍의 필요성에 대해 답변해주세요

**답변2.** 자바스크립트는 비동기 언어입니다. 동기는 코드가 순차적으로 실행됨을 의미하고, 비동기는 실행중인 코드가 완료 전에
다음 코드가 실행되는 동작을 의미합니다. 코드가 순차적으로 실행된다면 네트워크 요청, 파일 처리 등의 동작을 할 때
완료될 때까지 기다리게 되고, 다음 코드 실행 지연 시간이 크게 증가하게 됩니다. 따라서 이러한 동작에서는 비동기
프로그래밍을 통해 코드를 효율적으로 실행할 수 있게 설정할 수 있습니다.

**[Keyword]** 병렬 처리, 성능 개선

**질문3.** 브라우저의 작동방식에 대해서 설명해주세요

**답변3.** 브라우저 주소창에 URL을 입력하면, DNS에 도메인에 해당하는 IP 주소를 찾고, 해당 서버로 이동하게 됩니다.
서버에서는 클라이언트에서 보낸 http 요청을 처리하고 html,css,js등 리소스를 전달합니다. 이후 클라이언트 측에서 브라우저 렌더링이 실행됩니다.
브라우저 렌더링은 HTML 트리와 CSS 트리를 합쳐 렌더 트리를 생성하고, 해당 트리의 노드를 계산하여 배치(레이아웃)
하고 그리고(페인팅), 최종적으로 화면에 출력합니다. 이후 자바스크립트로 동적인 변화가 발생하면 레이아웃을 재배치하는 리플로우,
노드를 다시 그리는 리페인팅 과정이 발생합니다.

**[Keyword]** http, local host, TCP/IP, DOM, HTML, Script

**질문4.** GET, POST 방식의 차이점에 대해서 설명해주세요

**답변4.** GET 방식은 URL에 데이터를 포함하여 서버에 요청하며 데이터가 URL의 쿼리 매개변수로 전송되기 때문에
주소창에 노출됩니다. 따라서 보안에 취약하며 비밀번호와 같은 데이터들은 전성하지 않는 것이 좋습니다. 보통 검색어 전달이나
페이지 링크 클릭등에 사용됩니다.POST 방식은 HTTP 요청의 본문에 데이터를 포함하여 서버에 요청하여 URL에 노출되지 않고
요청의 본문에 포함되므로 비교적 안전한 편입니다. GET에 비해 상대적으로 보안이 높아서 민감한 정보를 안전하게 전달할
수 있습니다. 로그인 정보나, 폼에 작성된 데이터를 전송등에 사용됩니다.

**[Keyword]** 데이터 전송, URL, 보안

<hr />

**질문5.** GET, POST의 개념과 함께 데이터 흐름에 대해서 설명해주세요.

**답변5.** GET과 POST는 웹 서버와 클라이언트 간 데이터를 교환하는 HTTP 메서드입니다. GET은 주로 서버에서 정보를 조회하기 위해
사용하며, 요청하는 데이터는 URL의 쿼리 문자열에 포함되어 전송됩니다. 이로 인해 URL에 데이터가 노출되며, 크기 제한이
있습니다. POST는 데이터를 서버에 제출할 때 사용되며, 데이터는 요청 본문에 포함됩니다. 따라서 데이터의 크기 제한이 없으며,
민감한 정보 전송에 적합합니다. GET은 데이터 조회에, POST는 데이터 제출 및 변경에 주로 사용됩니다. 데이터 흐름에서
GET은 서버에서 클라이언트로, POST는 클라이언트에서 서버로 주로 흐릅니다.

**[Keyword]** 쿼리, 데이터, server, client

**질문6.** 쿠키, 세션, 로컬 스토리지의 차이에 대해 설명해주세요.

**답변6.** 쿠키, 세션, 그리고 웹 스토리지는 모두 클라이언트와 서버 사이에서 정보를 저장하고 교환하는 방법들이지만,
각각 목적과 작동 방식에 차이가 있습니다. 쿠키는 클라이언트 측에 저장되는 작은 데이터 조각으로, 사용자의 상태 정보나
ID를 저장하여 웹 서버가 사용자를 식별할 수 있게 해줍니다. 그러나, 쿠키는 보안에 취약하며, 저장할 수 있는 데이터 크기에
제한이 있습니다. 세션은 서버 측에서 관리되며, 클라이언트의 상태 정보를 서버에 저장합니다. 클라이언트는 세션 식별자(ID)만
쿠키로 저장하고, 이를 통해 서버는 클라이언트를 식별합니다. 이 방식은 쿠키보다 보안에 강하고, 데이터 제한이 적습니다.
웹 스토리지는 HTML5에 도입된 기술로, 로컬 스토리지와 세션 스토리지로 나뉩니다. 로컬 스토리지는 데이터를 클라이언트의
브라우저에 영구적으로 저장하며, 세션 스토리지는 브라우저 세션이 종료되면 데이터가 삭제되는 특성을 가집니다.
웹 스토리지는 상당히 큰 데이터를 안전하게 저장할 수 있으며, 서버로 데이터를 자동으로 보내지 않아 네트워크 효율이 높습니다.

- 쿠키는 클라이언트 측에 저장되는 작은 데이터 조각이며, 세션은 서버 측에서 사용자 정보를 관리합니다.
  웹 스토리지는 로컬 스토리지와 세션 스토리지로 나뉘며, 데이터의 보존 범위와 지속성이 다릅니다.

**[Keyword]** 저장 위치, 보안 강도, 유효 기간, 데이터 규모
