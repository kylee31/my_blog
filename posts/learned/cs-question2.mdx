**질문1.** var, let, const의 차이에 대해 알려주세요.

**답변1.** 세가지 명령어는 자바스크립트 변수 선언과 관련되어 있습니다. 각각의 명령어는 재선언과 재할당 방식에서
차이가 있는데요, var는 재선언, 재할당이 모두 가능합니다. ES6에서 새로 추가된 문법인 let, const는 재선언이 불가하나
재할당이 가능한 let과, 재선언, 재할당 모두 불가한 const가 있습니다. (스코프 차이: var 전역 스코프, let,const 블록
스코프로 엄격한 스코프 가짐, 호이스팅 차이: var는 호이스팅 되어 함수 최상단으로 undefined 초기화되어 끌어올려짐
(변수 할당값은 실제 코드 도달 시 할당), 선언 전 접근 가능 / let,const는 호이스팅에 초기화된 값이 들어가지 않음 TDZ.
선언 전 접근하면 참조 에러, 모두 선언문에서 초기화되며 const는 선언시 반드시 초기값 받아야 함)

**[Keyword]** 재선언/재할당, 호이스팅, 스코프

**질문2.** Async/Await와 Promise의 차이에 대해 설명해주세요.

**답변2.** 둘은 모두 비동기 작업과 관련된 문법입니다. 기본적으로 자바스크립트는 비동기 통신이기 때문에 이러한 문법을
활용하여 동기적인 작업이 가능합니다. Promise 이후 async/await 문법이 등장했습니다. promise는 성공,대기,실패 3가지
상태를 가지며 then, catch를 활용하여 비동기 작업이 끝난 후 실행할 로직을 구현할 수 있습니다. Async/await은 이러한
promise를 기반으로 합니다. 비동기 작업을 할 함수에 async를 선언하면 promise를 반환하고 await을 사용하여 Promise
결과가 반환될때까지 함수 실행을 일시 중지합니다. Try/catch를 활용하여 에러 처리가 가능힙니다. Async/await을 보다
직관적인 방법으로 가독성을 높이고 Promise 체인보다 선형적인 코드 흐름을 제공합니다.

**[Keyword]** 비동기 작업 처리, 에러 처리 방식

**질문3.** Arrow Function 이란 무엇인지 설명해주실 수 있을까요?

**답변3.** 화살표 함수란, 함수 작성법(함수 선언식, 함수 표현식, 익명 함수) 중 하나이며 익명 함수로 작성하는 함수입니다.
주로 함수를 즉시 실행해야 하는 상황이나 이벤트 핸들러의 콜백함수로 많이 활용됩니다.-해당 함수가 정의된 시점의 컨텍스트
유지가능하여 외부 스코프의 this값 활용가능 화살표 함수의 가장 큰 특징은 this가 없기 때문에, 해당 함수를 감싸는 외부
함수의 this를 상속받는다는 특징을 들 수 있습니다. (bind,call,apply-this컨텍스트를 명시적으로 사용할 때 쓰는 메소드를
사용하지 않아도 외부 함수 this 사용가능)

**[Keyword]** 익명 함수, 콜백 함수, this

**질문4.** ‘==’와 ‘===’ 연산자의 차이는 무엇인지 설명해주실 수 있을까요?

**답변4.** 동등 연산자와 일치 연산자의 차이점은 형변환 여부 및 타입 검사 여부입니다.
동등 연산자는 타입을 변환시켜 값의 일치 여부를 비교하고, 일치 연산자는 타입 변경 없이
원본 그대로의 일치 여부를 검사합니다. 예를 들어, number 1과 Boolean true를 비교할 때,
동등 연산자는 참을 반환하지만, 일치 연산자는 number와 boolean 타입은 다르기 때문에 거짓을 반환하게 됩니다

**[Keyword]** 값, 타입 비교

**질문5.** 데이터 10,000개를 가지고 무한 스크롤 구현시에 가장 중요하게 고려해야 할점은 무엇인가요?

**답변5.** 프로젝트 경험에 비추어 생각해보았을 때, 참조할 컴포넌트를 설정하는 것이 가장 중요하다고 생각합니다.
상품 리스트에 무한 스크롤 기능을 적용한 적이 있었는데요, 단순히 map함수로 반환하는 컴포넌트에 ref값을 참조되어
모든 컴포넌트가 참조되는 불필요한 동작이 일어났던 경험이 있습니다. 인덱스 값을 비교하여 가장 마지막 컴포넌트 값만
참조하도록 해당 부분에 조건문을 추가하며 코드를 수정하였던 경험이 있습니다.
네트워크 요청 역시 중요하게 고려할 점인데요, 참조 값을 관찰하고 마지막 컴포넌트 도달 시 true를 반환하고,
Firebase 쿼리를 활용하여 새로운 데이터를 개수를 지정하여 불러오도록 구현했습니다.

**[Keyword]** 가상화, 네트워크 요청, 렌더링, 참조

**질문6.** JSX란 무엇인가요?

**답변6.** . JSX란, React의 문법으로, Javascript와 HTML이 혼합된 문법입니다. 자바스크립트에서는 html과 javascript를
따로 작성해야하는데, React에서는 이러한 마크업과 로직을 JSX를 통해 결합하여 작성할 수 있으므로, 개발자가 코드 작성이
수월해지고 코드의 가독성을 향상시킬 수 있다는 장점을 들 수 있습니다. 그러나 JSX는 브라우저에서 직접 실행할 수 없기
때문에 바벨과 같은 트랜스파일러로 코드를 변환시켜야 합니다.

**[Keyword]** 마크다운/HTML, 트랜스파일러
