> 디바운싱 (Debouncing):
> 디바운싱은 연속적으로 발생하는 이벤트가 일정 시간 동안 없을 때에만 이벤트를 처리하는 기술입니다.

> 쓰로틀링 (Throttling):
> 쓰로틀링은 연속적으로 발생하는 이벤트의 처리 빈도를 제한하는 기술입니다.

디바운싱과 쓰로틀링은 둘 다 성능 및 사용자 경험을 향상시키기 위해 사용되는 기술입니다.
둘 다 이벤트 핸들러를 제어하여 연속적으로 발생하는 이벤트의 처리를 최적화합니다.

디바운싱 (Debouncing):
디바운싱은 연속적으로 발생하는 이벤트가 일정 시간 동안 없을 때에만 이벤트를 처리하는 기술입니다.
예를 들어, 사용자가 텍스트를 입력할 때마다 검색 요청을 보내는 경우를 생각해보겠습니다.
사용자가 글자를 입력할 때마다 검색 요청을 보내면 서버에 부하가 걸리고, 응답을 받기 전에 또 다른 검색 요청이 발생할 수 있습니다.
이런 경우 디바운싱을 사용하여 사용자가 일정 시간 동안 입력을 멈추었을 때에만 검색 요청을 보내도록 설정할 수 있습니다.
이렇게 함으로써 불필요한 요청을 줄이고 서버 부하를 감소시킬 수 있습니다.

쓰로틀링 (Throttling):
쓰로틀링은 연속적으로 발생하는 이벤트의 처리 빈도를 제한하는 기술입니다.
일정 시간 동안에 여러 번 발생하는 이벤트 중에서 일정 시간 간격으로 하나의 이벤트만을 처리하도록 설정합니다.
예를 들어, 사용자가 스크롤을 할 때마다 특정 작업을 수행하는 경우를 생각해보겠습니다.
사용자가 빠르게 스크롤을 할 경우 너무 많은 이벤트가 발생하여 처리할 수 없을 수 있습니다.
이런 경우 쓰로틀링을 사용하여 일정 시간 간격으로만 스크롤 이벤트를 처리하도록 설정할 수 있습니다.
이렇게 함으로써 사용자 경험을 향상시키고 불필요한 작업을 방지할 수 있습니다.

디바운싱과 쓰로틀링은 각각 다른 상황에 적합하게 사용됩니다.
**디바운싱**은 이벤트의 **발생 빈도**를 줄이는 것에 중점을 두며, **쓰로틀링**은 이벤트의 **처리 빈도**를 제한하는 것에 중점을 둡니다.
이 두 기술은 모두 프론트엔드에서 성능 및 사용자 경험을 향상시키는 데 유용하게 사용됩니다.

**💻 예제**

- 디바운싱 : 연이어 발생하는 이벤트 중 마지막 이벤트에 대해서만 함수를 호출하는 기술<br/>
  [예제] 검색어 입력창에서 사용자가 타이핑할 때, 디바운싱을 사용하면 마지막 입력 이벤트 후에만 실제 검색을 실행할 수 있다.

```js
import { debounce } from "lodash";

const debouncedFunction = debounce(() => {
  console.log("디바운싱 중!");
}, 1000);

inputElement.addEventListener("input", debouncedFunction);
```

- 쓰로틀링 : 특정 동작이 주어진 시간 동안 최대 한 번만 발생하도록 하는 기술<br/>
  [예제] 사용자의 스크롤 이벤트에 대응해 특정 함수를 호출할 때, 쓰로틀링을 사용하면 연속적인 스크롤에 대한 함수 호출을 제어할 수 있다.

```js
import { throttle } from "lodash";

const throttledFunction = throttle(() => {
  console.log("쓰로틀링 중!");
}, 1000);

window.addEventListener("scroll", throttledFunction);
```
