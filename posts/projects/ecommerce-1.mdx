> 커머스 플랫폼 기획 및 프로젝트 초기 환경구성 단계

프로젝트 시작 전 Figma를 활용하여 와이어프레임을 작성하였다. 메인, 로그인/회원가입, 판매자, 구매자, 카테고리, 상품 상세
페이지를 기획하여 레이아웃 배치해보았다.

![](/projects/ecommerce/1-1.jpg)

UI 제작과 더불어 유저 플로우도 함께 표로 작성했다. 커머스에서 중요하게 생각할 부분은 사용자가 어떤 권한을 가지는지, 권한에
따라 어떤 페이지 및 기능을 사용할 수 있는지 생각해야했다.

사용자 부류는 총 3가지로 접근자, 구매자, 판매자로 나누어 생각했다.

- 접근자 : 회원가입 없이 웹 사이트에 접속하여 사용하는 이용자
- 구매자 : 구매자 계정으로 로그인하여 웹 사이트를 사용하는 이용자
- 판매자 : 판매자 계정으로 로그인하여 웹 사이트를 사용하는 이용자

초기 기획에서 로그인 하지 않은 사용자라도 상품을 장바구니에 담을 수 있어야 UX가 향상된다고 생각했기 때문에 홈 화면, 카테고리,
상품 상세 페이지 및 장바구니 버튼 클릭까지 모두 이용가능한 범위로 두었다. 아래는 로그인 여부에 따라 작성한 유저 플로우이다.

- 구매자
  ![](/projects/ecommerce/1-2.jpg)

- 판매자

이렇게 초기 유저 플로우, UI 와이어프레임 생성 후 프로젝트 초기 환경세팅을 진행했다. 프로젝트에 사용될 기술 스택은 다음과 같다.

- React
- Typescript
- Tailwind CSS (Shadcn/UI) : 사용자에게 미리 정의된 유틸리티 클래스들을 제공하는 CSS 프레임워크
- Firebase (cloud storage, store, auth)
- Vite : 가볍기 때문에 속도가 빠르고 성능이 향상된 빌드 도구 (일부 번들러 기능)

Vite를 활용하여 React, Typescript 스펙을 설정하여 프로젝트를 빌드하였다.

```js
npm create vite@latest
```

```
Project Name: ./
Select a framework: React
Select a variant: Typescript
```

이후 프로젝트에 Tailwind CSS를 설치하여 초기 프로젝트 세팅을 완료하였다.

[1주차 구현 요구사항]

- ✅ 페이지 라우팅 설계
- ✅ 로그인 / 회원가입
- ✅ 판매자 (상품 CRUD)
