> 컴포넌트 구조를 재설계한 후 페이지 컴포넌트(서버 컴포넌트)에 SEO 최적화를 위해 metadata 작성

아직 SEO 최적화를 위해 학습이 필요하지만, 기본적으로 적용해보고 싶었다!

```js
export const metadata: Metadata = {
  title: "TO TRAVEL IS TO LIVE!",
  description:
    "지역을 검색하여 추천 장소를 알아보고, 캘린더 기록 및 메모 작성이 한 페이지 내에서 가능한 여행(국내) 계획 세우기 서비스",
  keywords: ["Next.js", "국내 여행", "토이 프로젝트"],
  //추가 학습 후 새로 작성 필요
  // viewport: "initial-scale=1.0, width=device-width",
  // openGraph: {
  //   images: "/imgs/travel.png",
  //   type: "article",
  //   locale: "ko_KR",
  // },
};
```

metadata를 각각 로딩 페이지, 메인 페이지에 맞게 작성했다. 제목과 설명, 키워드 정도만 다르게 작성해보았고, 페이지 변경시 html 정보가
변경됨을 확인할 수 있었다.

- 로딩 페이지 keywords
  ![](/projects/travelplan/5-1.jpg)

- 메인 페이지 keywords
  ![](/projects/travelplan/5-2.jpg)

기본적인 metadata를 작성하였으니, 다른 viewport, openGraph 속성 등을 설정하는 방법을 학습해야겠다.

- viewport : 모바일 기기에서 웹 페이지를 표시할 때 화면 크기와 비율을 제어하는 태그
- Open Graph : 웹 페이지가 소셜 미디어에서 공유될 때 미리보기에 사용되는 메타데이터를 정의하는 태그

또 페이지 별 하드코딩이 아닌, layout 페이지에서 동적으로 적용하는 방법을 추가적으로 알아보기로!
