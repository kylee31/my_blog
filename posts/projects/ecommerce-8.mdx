> vitest를 사용하여 간단한 유틸리티 코드에 대해 테스트 코드를 작성했다. 통합 테스트, E2E 테스트에 대해 더 알아보고 싶어서 진행하던 커머스
프로젝트에 계속 작성해보는 중!

👩‍💻 로그인 로직에 대한 테스트 코드를 작성했다. 유틸리티는 순수 함수라 expect와 toBe만 사용하여 작성했는데 로그인 로직은 실제 Firebase에
접속하는 로직이라 mock 함수를 생성하여 진행할 수 있었다.

▶ 여기서 짚고 넘어갈 것. 나는 로그인 로직이기 떄문에 단순히 통합 테스트라고 생각했다. (외부 환경과의 통신이 있기 때문에) 그러나 다시 생각해보면
목 함수이므로 실제 외부 환경과의 통신이 일어나지 않는다는 것을 알 수 있다.

- 단위 테스트 (Unit Test)

    목적: 코드의 개별 단위를 테스트하여 해당 단위가 올바르게 동작하는지 확인.
    방법: 외부 종속성(예: 데이터베이스, 네트워크, 파일 시스템)을 모킹하여 테스트의 독립성을 유지.
    특징: 빠르고, 독립적이며, 종속성 주입을 통해 특정 부분만 테스트 가능.

- 통합 테스트 (Integration Test)

    목적: 시스템의 여러 구성 요소가 함께 올바르게 작동하는지 확인.
    방법: 실제 서비스나 데이터베이스와의 상호작용을 포함하여 전체 시스템의 통합을 테스트.
    특징: 실제 환경에 가까운 시나리오를 테스트하며, 종종 느리고 복잡할 수 있음.

결론은! 목 함수를 사용했기 때문에 단위 테스트에 가깝다~
