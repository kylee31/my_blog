> 커머스 프로젝트 리팩토링 및 성능 최적화

**✨ 짧은 회고**

기술 매니저님의 조언에 따라 리팩토링 과정에서는 기능 수정 및 추가를 하지 않았다. 리팩토링은 단어 그대로 코드의 가독성 및
구조 향상을 위해 수정하는 것이지 기능을 수정하는 과정이 아니니까, 로직에 신경 쓰라는 조언을 주셨다.

아직 블로그에 정리 하진 못했지만, 비즈니스/뷰 로직 분리에 중점을 두고 프로젝트를 진행하니 이전보다 시야가 넓어진 것 같았다.

독학으로 학습할 당시에는 실습을 통해 단순히 기술스택을 많이 다뤄보고, CS 기초를 탄탄히 해야겠다는 생각이 컸다.
이후 스타트업에서 근무하며 짧게나마 웹 개발 전체 사이클(기획-1차 배포)을 경험할 수 있었고, 프론트엔드 개발자로서 팀원과 어떻게
협업해야 하는지, 코딩 외에도 어떤 부가적인 업무를 맡게 되는지.. 스타트업의 특성에 대해서도 추가적으로ㅎㅎ 습득했다.
작은 규모지만 뛰어난 팀원 분들이라 작은 성과라도 내서 기여하고 싶었지만 그저 배우기만 했던 것 같다. 할 수 있는 건
적극적으로 참여하고, 학습했던 일 뿐😂

그렇기 때문에 새로 진행되는 2차 프로젝트에 참여하지 않았다. 이렇게 쫓아가기만 한다면 분명 이 직업에 금세 지칠 것 같았다.
가고 싶었던 기업 공고가 올라와서 면접까지 보았지만, 탈락했고 부족한 점을 많이 느끼며 그때문에 더 혼란했던 것 같다.
그렇게 12월은 짧은 휴식기였고, 1~2월동안 나에게 적합한 부트캠프가 있는지 찾아보며 블로그를 새로 만들었다.

짧은 기간동안 진행되며, 현직 개발자인 기술 매니저님, 멘토님과의 소통이 있는 프로그램을 찾았고 합류하게 되었다.
누군가에겐 별 것 아닐 수 있지만 프로젝트를 진행하며 얻었던 조언과 팁들이 내 코드를 향상시키는데 있어 많은 도움이 되었다.

(추가로, 한번에 원했던 기업을 통과했던 이력서 역시 운이었다는 사실을 깨달았다.. 해당 기업에만 적합한 자유 양식이였던 것..ㅜㅜ)

아무튼! 리팩토링 전 많은 걸 얻어 코드 개선이 되었다는 사실을 기록하고 싶어서 이 게시글에 작성해둔다ㅎㅎ

<hr />

4주차는 구현한 코드를 리팩토링하며 성능 최적화를 하는 시간을 가졌다. 이전에도 간간히 리팩토링을 했기 때문에 어떤 것에 중점을
두었는지 적어보면 다음과 같다.

- 비즈니스/뷰 로직 분리
- 이미지 최적화
- 렌더링 최적화
- 번들 크기 축소
- 데이터 캐싱
- UI 개선 (Lighthouse Performance CLS 성능 개선)

추가로 UX 향상을 위해 판매자 상품 등록 시 실시간으로 데이터 변경을 감지하여 반영하는 동작을 구현하고 싶었다.
이는 Firebase onSnapShot을 활용하여 DB를 구독하고, 변경사항이 생기면 React-Query의 enabled 옵션을 조작하여 쿼리를 실행하도록
구현해보았다. (이에 대한 트러블슈팅은 따로 빼서 작성 중..!)
