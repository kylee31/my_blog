> Vercel에서 제공하는 배포 기능을 이용하여 Github repository에 올린 프로젝트를 배포해보자.

이전 프로젝트에서 AWS EC2를 통해 인스턴스를 받아 서버에 배포하는 과정(클라우드 웹서비스, 주소와 도메인 연결, ssh키 등록,
Github Action 연결로 빌드 시 자동배포 등)을 하나하나 직접 경험해보았기 때문에 배포 과정에 대한 전체적인 내용을 이해할 수 있었다. 주소창에 URL을 입력하면
어떤 흐름으로 서버에 요청을 보내고 응답 받는지 확인할 수 있었다. 이번에도 Next.js로 구현했기 때문에 배포 방법으로
조금 더 편리한 Vercel을 선택해보았다.

아래는 배포 과정에 대한 정리이다. 매우 쉬우니 참고만 하자!

<hr />

1. 배포를 위해 먼저 Vercel에 가입한다. (Github 계정으로 가입)
2. start deploying 선택한다.
3. Github 사용이므로 Continue with GitHub을 선택하고, Import Git repository 페이지로 접속한다.
4. import할 레포를 선택한다.
5. 가져온 레포에서 import를 선택하면 Configure Project라는 deploy 설정 페이지가 나온다.
   ![](/projects/myblog/4-1.jpg)

   이 단계에서 Project Name, Framework Preset, Root Directory를 설정하고 아래 세팅에서 빌드, 아웃풋,
   환경변수 설정이 가능하다.

   - bulid and output settings에서 Bulid Command를 설정해준다 (next build)

6. 위의 단계에서 여러 설정을 완료 후 deploy하면 다음과 같이 배포된다!
   ![](/projects/myblog/4-2.jpg)
7. 대시보드에서 발급된 도메인(주소)을 확인하고 사용하면 된다.

### 💻 [my blog](https://twothreeone-blog.vercel.app/)<br/>

매우 간단한 과정으로 Next.js에서 프로젝트를 build해주고 배포까지 해준다.
연동된 GitHub repo의 branch에 push될 때마다 자동으로 Vercel에서 배포해주게 된다.
