### 좋은 Readme 작성법

- 각 프로젝트 별 Readme를 작성해두었지만 일반 Readme가 아닌 포트폴리오에 가까운 내용을 작성해두었다는 점을 깨달았다. 좋은 Readme를 작성하기 위해 [infograb-좋은 Readme 작성법](https://insight.infograb.net/blog/2023/08/23/good-readme/) 참고, 실제 오픈소스 라이브러리의 Readme(쿠버네티스 등)를 참고하여 수정해야겠다고 느꼈다.

2. 호스팅 주소 삽입? 프로젝트 매핑하기
3. as-is, to-be 문서로 작성해서 최종 프로젝트를 위한 docs를 따로 빼서 넣기 (gitbook)

들어가면 좋은 내용: 주차별 요구사항, 트러블 슈팅, 설계 관련 고민들, 하고 싶은 얘기
