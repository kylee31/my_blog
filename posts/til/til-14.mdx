**질문1.** 쓰로틀링과 디바운싱의 개념과 사용하는 이유에 대해서 설명해주세요

**답변1.** 쓰로틀링은 특정 동작을 일정 시간동안 한번만 발생하도록 횟수 제한하는 기법입니다, 디바운싱은 연속 동작 중
마지막 이벤트만 호출하는 하는 기법입니다. 자세한 예를 들자면 쓰로틀링은 scroll이 빠르게 연속으로 발생하는 것에 대비하여
일정시간 동안 한번만 일어나도록 설정할 수 있습니다. 디바운싱은 input으로 값을 입력받을 때 연속적으로 요청을 보내는 것이
아닌 더 이상 입력이 발생하지 않았을 때, 즉 마지막 입력이 끝나면 요청하도록 할 수 있습니다. 두 기법 모두 불필요한
동작을 줄여 예상치 못한 동작을 방지하고 성능을 최적화하기 위해 사용합니다.

**[Keyword]** 성능 최적화, 이벤트, 사용자 경험 개선, 안정성

**질문2.** 자바스크립트와 타입스크립트의 차이, 장/단점에 대해 설명해주세요.

**답변2.** 자바스크립트와 타입스크립트는 실행 과정과 타입 안정성에서 차이점이 존재합니다. 자바스크립트는 정적 타입 검사가
없기 때문에 자유도가 높다는 장점이 존재하지만 동시에 타입 안정성이 낮습니다. 타입스크립트는 자바스크립트의 슈퍼셋으로,
정적 타입 검사와 컴파일 과정을 거쳐 자바스크립트로 변환되어 실행됩니다. 타입스크립트에서는 값의 타입을 지정하여 사용되는
데이터를 정확하게 다룰 수 있다는 점이 장점이지만, 타입의 자유도가 그만큼 낮아지는 점이 단점입니다. 따라서 작은 규모에는
자바스크립트가 적합할 수 있지만 유지보수 측면에서 타입스크립트가 적합합니다.

**[Keyword]** 브라우저, 타입, 코드 가독성과 유지 보수, 컴파일

**질문3.** SEO란 무엇이고 SEO를 진행하는 방법에 대해 설명해주세요.

**답변3.** SEO란 검색 엔진 최적화로, 웹 사이트가 메타 데이터를 활용하여 검색 엔진에 효과적으로 노출될 수 있도록 다양한
요소를 최적화하는 과정을 의미합니다. 예를 들어 SSR 방식을 적용하여 웹 사이트에 메타 태그를 적용하면, 해당 데이터를 검색
엔진 봇이 크롤링하게 됩니다. 수집한 값을 기반으로 검색 시 사용자에게 노출해주는 방식을 활용할 수 있습니다. SEO 최적화를
다양하게 적극적으로 적용해본 경험은 없지만, Next.js의 SSR 방식을 적용한 로딩 페이지와 메인 페이지 컴포넌트에 metadata로
제목, 설명, 키워드를 작성해본 경험이 있습니다.

**[Keyword]** 최적화, 키워드, 백링크, 유입과 이탈

**질문4.** 이미지 최적화에 대해 설명해주시고 방법에 대해 설명해주세요.

**답변4.** 제가 생각하는 이미지 최적화란 크게 이미지 lazy loading을 통한 최적화와 이미지 사이즈 압축을 통한 최적화,
이 2가지 방식이 있는 것 같습니다. 이미지 태그의 loading 속성을 활용하여 이미지에 도달하였을 때 로딩하는 기법으로
처리할 수 있습니다. 이미지 사이즈 최적화는 정적인 경우와 동적인 경우로 나눌 수 있을 것 같습니다. 정적인 경우는 파일
포맷을 jpg, png 대신 webp 형식을 사용하여 최적화할 수 있습니다. 이미지 리사이징 라이브러리를 설치해서 파일 업로드 시
이미지를 상태에 추가하기 전 압축 후 상태를 업데이트 하는 방식으로 최적화를 할 수 있습니다.

**[Keyword]** 압축, 리사이즈, 파일 확장자, 지연 로딩

**질문5.** 프론트엔드 개발 시 개발자 도구를 활용한 경험이 있다면 설명해주세요.

**답변5.** 개발 시 저는 주로 개발자 도구에서 콘솔창, 네트워크 탭, Application, Lighthouse을 사용하는데 각각
사용 목적이 달랐습니다. 먼저 콘솔창은 주로 개발 중 데이터, 라이프 사이클, 에러를 확인할 때 사용하였습니다.
네트워크 탭에선 서버와 데이터 통신을 할 때, header에 제가 담은 데이터들이 정상적으로 가는지와 response를 어떤
식으로 받아오는지 확인할 수 있었습니다. Application에선 주로 로컬 스토리지에 토큰이 제대로 담기는지, 토큰(uid)
데이터가 정상적으로 삭제되는지를 확인하는 용도로 사용했습니다. lighthouse는 어느정도 프로젝트가 진행되고 난 후에
성능 최적화를 위해 사용했는데, performance와 SEO등을 확인해서 어떤 부분에서 로드가 오래 걸리는지, 이를 바탕으로
최적화를 위해 가장 먼저 수정해야 할 것이 무엇인지 등을 고려할 때 참고자료로서 사용했습니다.

**[Keyword]** 콘솔창, 네트워크탭, application, Lighthouse

**질문6.** 프론트엔드 성능최적화를 위한 방법이 있다면 말씀해주시고 그 중 프로젝트에서 활용한 경험이 있다면 설명해주세요.

**답변6.** 성능 최적화 방법으로는 이미지 최적화, 코드 스플리팅, 불필요한 렌더링 방지 등을 예로 들수 있고,
앞의 예시들을 적용해본 경험이 있습니다. 먼저 이미지 최적화를 위해 정적 이미지의 경우 jpg나 png가 아닌 webp 포맷을
사용해서 용량을 최소화 하였고 동적 이미지일 경우 라이브러리를 활용해서 용량을 압축했습니다. React 18부터 사용할 수
있는 Suspense api와 React.lazy와 같은 코드 스플리팅 방식을 통해 앱의 퍼포먼스를 크게 향상시킬 수 있었습니다. Vite를
활용하여 rollup 옵션으로 chunk를 분리하기도 했습니다. React DevTool을 사용하여 렌더링을 확인하고, useMemo를 사용하여
컴포넌트의 불필요한 렌더링을 줄였던 경험이 있습니다.

**[Keyword]** 압축, 캐싱, 비동기 처리, 지연 로딩, 렌더링 개선
